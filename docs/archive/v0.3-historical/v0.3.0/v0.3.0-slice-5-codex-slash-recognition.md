## 1. Background & Goal

**Background.** `alaya attach codex` writes an Alaya-managed
`[slash_commands.alaya-inspect]` profile entry that launches `node
<repo>/bin/alaya.mjs inspect --open`. Current project truth proves
the profile mutation and the CLI launcher shape, but not that the
active Codex CLI version actually recognizes that custom slash
registry inside the conversation composer. `#BL-037` requires
closing that gap one way or the other.

**Goal.** Either prove that the active Codex CLI recognizes the
`/alaya-inspect` registry entry (and, if the documented format
differs from what `attach` writes, update the writer to the
documented format with the same audit-first, preview-then-confirm
hygiene), or document that Codex does not support third-party fixed
slash triggers in the tested version and retire the `cli-consumable`
expectation from `runtime-status.md`. Either outcome closes
`#BL-037`.

- **Action**: determine the supported Codex extension path for a

    supported by host" with a `docs/handbook/maintenance.md` note
    citing the Codex version tested and the supported fallback path.

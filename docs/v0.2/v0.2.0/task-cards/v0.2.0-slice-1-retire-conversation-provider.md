## 1. Background & Goal

**Background.** The engine-gateway package carries three placeholder
symbols ported from `do-what-new` that have never had a production
caller in Alaya. `provider-registry.ts:10-16` `resolveLanguageModel()`
throws `EngineError("…deferred to #BL-008")` unconditionally;
`provider-types.ts:33` `ConversationProvider` has zero cross-package
imports in the repository; `daemon-runtime-support.ts:793-805`
`createAlayaConversationEngine()` returns a hardcoded string
"Alaya does not execute chat turns; use MCP memory tools." The
daemon wires this stub at `apps/core-daemon/src/index.ts:604-606`
through a `Record<string, unknown>` widening cast at line 623 because
`ConversationServiceDependencies` does not declare an `engine` field.
Invariant §21 already retired the chat-turn use case the interface
modeled. Three architectural reviewers concurred on deletion (D'
stance) in the planning phase.

**Goal.** Delete the dead `ConversationProvider` /
`resolveLanguageModel` / `createAlayaConversationEngine` surface and
its orphan wiring; preserve `EngineBinding`, `readApiKey`,
`resolveApiKey`, and `createEngineBindingTester` (live
infrastructure); record a one-paragraph closeout note in
`docs/handbook/backlog.md`.

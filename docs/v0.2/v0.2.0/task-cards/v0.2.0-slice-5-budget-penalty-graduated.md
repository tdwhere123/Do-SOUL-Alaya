## 1. Background & Goal

**Background.** `mapBudgetPenalty` at
`packages/core/src/recall-service-helpers.ts:164-175` is a hardcoded
switch — `BankruptcyKind.NONE -> 0`, `SOFT -> 0.3`, `HARD -> 1`. The
penalty multiplies the `budget_penalty` weight inside
`computeEffectiveScoreDetails` and shifts recall ordering when the
workspace is under budget pressure. The static switch is coarse: a
soft-bankruptcy at 51% utilization and one at 99% receive identical
penalty.

**Goal.** Replace the switch with a graduated function reading a
new `pressure_ratio: number ∈ [0, 1]` field on `BudgetSnapshot`.
Boundary anchors stay (`NONE -> 0`, `HARD -> 1`); `SOFT` interpolates
monotonically with the ratio. Pin behaviour at five representative
ratio points with a unit test.

## 2. Allowed Scope

### packages/protocol/src/soul/budget-snapshot.ts

- **Change**: extend `BudgetSnapshotSchema` with a new
  **optional / defaulted** field
  `pressure_ratio: z.number().min(0).max(1).default(0).readonly()`.
  It is NOT a required field: `BudgetSnapshotSchema` is exported
  from `@do-soul/alaya-protocol`, and §25 classifies a new required
  field as a major bump — a sibling consumer parsing or constructing
  an old snapshot must still validate. With `.default(0)` the schema
  is backward-compatible (an old snapshot without the field parses,
  yielding `0`), so this stays an additive minor. Producers always
  emit the real value (see below) — the default exists only for
  parse-compatibility, not as a runtime fallback the producers rely
  on.
- **Change**: if the file exports a `BudgetSnapshot` TS type alias
  derived from the schema, the type widens accordingly (the property
  becomes present-with-default after parse); no consumer-facing
  breaking change.

### BudgetBankruptcyService + daemon budget wiring

- **Change**: locate every place that builds a `BudgetSnapshot`
  (`rg -n 'class BudgetBankruptcyService' packages/core/src`, plus
  `apps/core-daemon/src/budget-wiring.ts` and any other snapshot
  producer). Each computes `pressure_ratio` as
  `tokens_used / max_total_tokens` (or the equivalent utilization
  formula already informing `bankruptcy_kind`), clamped to `[0, 1]`.
  The existing `bankruptcy_kind` derivation stays — graduated
  penalty is additive, not a replacement. A producer-completeness
  check (test or grep assertion) verifies no snapshot producer is
  left emitting the bare `.default(0)`.

### packages/core/src/recall-service-helpers.ts

- **Change**: rewrite `mapBudgetPenalty(snapshot:
  Readonly<BudgetSnapshot>): number` as a graduated function. Use
  the following monotonic curve as the v0.2.0 default:
  - `NONE` (i.e. `pressure_ratio < softThreshold`): returns 0
  - `SOFT` (`softThreshold ≤ pressure_ratio < hardThreshold`):
    returns `0.1 + 0.6 * (pressure_ratio - softThreshold) /
    (hardThreshold - softThreshold)` — yielding ~0.1 at the soft
    edge and ~0.7 at the hard edge
  - `HARD` (`pressure_ratio ≥ hardThreshold`): returns 1
- `softThreshold` and `hardThreshold` constants stay in the
  protocol's dynamics constants module (or wherever
  `bankruptcy_kind` was previously derived). Do not introduce new
  configuration knobs — the curve is the v0.2.0 baseline; future
  tuning can replace the constants.
- **Keep** the existing caller `recall-service.ts:544-547`
  unchanged; the signature shape is preserved.

### Unit test (locate `recall-service-helpers.test.ts` or add)

- **Change**: a new test pins behaviour at five
  `pressure_ratio` points: 0.0, 0.4 (mid-NONE), 0.6 (low-SOFT), 0.85
  (high-SOFT), 1.0. The expected return values follow the curve
  above. The test stays valid as long as the curve is monotonic and
  the boundary anchors are preserved; a future revision must update
  this test deliberately.

## 3. Deferred

- Per-domain budget penalty tuning is deferred to v0.2.0-slice-7
  (per-domain weights). Today the penalty is global, after Slice 7
  the *weight* applied to the penalty becomes per-domain — but the
  penalty value itself remains global. No backlog entry is opened; the
  global penalty curve is the current contract.

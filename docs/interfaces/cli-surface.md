# CLI Surface

The CLI is for local operation, diagnostics, backup, and scripted use. This
ignored prototype is not a packaged install target yet.

## Command Shape

Current local prototype binary:

```bash
rtk node dist/cli/index.js
```

The future packaged target can remain `soul-memory`, but this ignored prototype
is run directly from `dist/cli/index.js` after TypeScript build.

Implemented prototype commands:

```bash
soul-memory setup
soul-memory serve
soul-memory doctor
soul-memory recall
soul-memory ingest
soul-memory context
soul-memory session
soul-memory govern
soul-memory backup
soul-memory export
soul-memory import
soul-memory mcp config
soul-memory gateway -- <agent command>
soul-memory inspector
```

## Command Responsibilities

### `setup`

Create local config, storage directories, default profile, and optional agent
MCP config snippets.

### `serve`

Start the local HTTP API and inspector asset server. MCP is exposed as a
separate stdio entrypoint generated by `mcp config`; `serve` should print that
MCP mode is `stdio`, not imply an MCP listener is already running.

### `doctor`

Current prototype checks storage, baseline schema migration, and standalone
runtime wiring. Permission checks, MCP availability checks, and known agent
config checks belong to the future packaged product.

### `inspect`

Future terminal inspection surface. The current prototype uses `recall`,
`context`, `export`, and the HTML inspector.

### `recall`

Run recall from the shell for debugging and benchmarks.

### `ingest`

Write a manual note or evidence file into memory. Requires source and scope.

### `audit`

Future shorthand for memory events and governance changes. The current
prototype exposes audit through export, HTTP, MCP, and inspector paths.

### `backup` / `restore`

Portable local archives. The current prototype implements `backup`; restore is
future work.

### `export` / `import`

Interchange format for moving scoped memory between machines or products.

### `mcp config`

Print MCP config for a target agent. The current prototype does not mutate agent
config files.

### `gateway`

Gateway Mode entrypoint for agents that should be forced through SOUL Memory.
It should:

- start or locate the local daemon;
- start a memory session;
- assemble and attach a context pack;
- launch the selected agent;
- record memory usage events where observable;
- request or perform post-run ingest;
- finish the memory session with missed-step and violation data.

### `inspector`

Open or print the local HTML inspector URL.

## Current Run Rule

Run the local prototype directly. Do not rely on hidden postinstall side effects
or global package state.

Current prototype:

```bash
rtk pnpm exec tsc -p tsconfig.json
rtk node dist/cli/index.js setup --data-dir /tmp/soul-memory-product
rtk node dist/cli/index.js doctor --data-dir /tmp/soul-memory-product
rtk node dist/cli/index.js serve --data-dir /tmp/soul-memory-product
rtk node dist/cli/index.js mcp config --agent codex --data-dir /tmp/soul-memory-product
rtk node dist/cli/index.js gateway --data-dir /tmp/soul-memory-product --query "task context" -- node -e "process.exit(0)"
```

Future packaging and global install are intentionally outside this ignored
prototype and should move with the later standalone product folder/repository.

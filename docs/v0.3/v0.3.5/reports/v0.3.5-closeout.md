## Delivered

- CLI shim hardening: root and packaged `bin/alaya.mjs` use fixed
  module loader slots for bridge/register/daemon modules while
  preserving explicit test injection.
- MCP stdio startup diagnostics: workspace/run/session/bootstrap
  failures are reported on stderr with software exit code; stdout stays
  reserved for JSON-RPC frames. Background services start only after
  stdio server creation succeeds.
- Background logging hardening: `BackgroundServiceManager` accepts an
  injected logger and Garden runtime passes the daemon warn logger
  through background service and runtime warning paths.
- Local execution support:
  - `tools.exec_shell` remains `execFile`/argv-based and now has
    regression coverage for env filtering, non-shell argument handling,
    non-zero exits, and timeouts.
  - Environment status tool probes resolve executables directly from
    `PATH` instead of using a shell and keep worktree counting bounded
    by timeout and env allowlist.
  - Keychain subprocesses run with an allowlisted child environment.
- Install and metadata closeout: README install flow leads with a
  tag-pinned installer fetch, workspace packages are aligned to
  `0.3.5`, and root `package.json` has `version: 0.3.5` while staying
  private.

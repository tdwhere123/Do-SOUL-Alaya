## Parent Synthesis

The strongest current conclusion is: **this worktree should not be accepted as merge-ready at HEAD `539e32b`.** The recall fan-in implementation has plausible production wiring, and I did not find a direct fan-in correctness blocker, but its strongest delivery test is synthetic and #5b is still missing. The forgetting lifecycle has stronger problems: reversible dormancy mutates recall visibility without per-memory audit, terminal GC is implemented but not scheduled, and the compress arm has both missing producer wiring and protection/TOCTOU risks if activated.

The host_worker / zero-cloud default itself looks substantially correct from the inspected code:

- default Garden compute resolves to `host_worker` when no secret is configured;
- cloud edge-LLM is strict opt-in behind `ALAYA_EDGE_PRODUCER_LLM_ENABLED`;
- no hardcoded yunwu/provider URL was found on the edge/reconciliation LLM paths;
- doctor/status count pending and stale `EDGE_CLASSIFY` tasks.

That positive result does not clear I2: the EDGE_CLASSIFY completion envelope can still record false success when the verdict is missing.

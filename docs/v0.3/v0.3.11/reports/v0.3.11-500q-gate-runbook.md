# v0.3.11 500q R@5 Gate — Operator Runbook (run on a larger host)

The v0.3.11 closeout is code-complete and merged to `main`; the only remaining
release gate is the **big-machine 500q recall-quality run** (the local 7.6 GB
WSL2 box OOMs at 500q). This runbook is self-contained: on a fresh machine,
`git pull` then follow the steps below. **Do not tag `v0.3.11` until K1.1
passes here.**

## What `git pull` already gives you (no transfer needed)
- All code + the bench-runner (`apps/bench-runner`).
- The orchestrator script `scripts/run-full-bench-v0311.mjs` (its file header is
  the authoritative usage reference).
- The KPI targets (`docs/v0.3/v0.3.11/kpi-targets.md`) and the closeout records
  (`reports/v0.3.11-completion-masterplan.md`, `reports/v0.3.11-closeout-report.md`,
  `reports/v0.3.11-independent-multilens-review-2026-06-04.md`).
- The **tracked** LongMemEval extraction cache (~1138 fixtures under
  `docs/bench-history/datasets/longmemeval-extraction-cache/`) — covers roughly
  the first ~100 questions for free; the remaining ~400 of the 500q run are
  filled on the fly (see Cost below).

## What you MUST transfer manually (gitignored — `git pull` cannot bring it)
Copy these two from the source machine to the same paths on the big host:
1. **Secrets** `.do-it/bench-env/alaya-api.env` (~626 B). Holds the garden API
   key + provider URL + model (`ALAYA_OFFICIAL_GARDEN_API_KEY` /
   `OFFICIAL_API_GARDEN_PROVIDER_URL` / `OFFICIAL_API_GARDEN_MODEL=gpt-5.4-mini`),
   plus the OpenAI embedding key (only needed for the embedding-ON K1.5 wave,
   not the core gate). If you prefer, set the same env vars by hand instead of
   copying the file.
2. **Datasets** `apps/bench-runner/data/` (~268 MB; `locomo` + `longmemeval`).
   Either `scp`/rsync this directory, or re-acquire the public LongMemEval-S +
   LoCoMo datasets into the same layout.

(Transferring just `.do-it/` is NOT enough — it does not contain the 268 MB
dataset, which lives under `apps/`.)

## Setup
```bash
cd <repo>
git checkout main && git pull
rtk pnpm install      # or: pnpm install
rtk pnpm build        # tsc -b across the workspace; required before the bench runs
```

## Run the gate
```bash
# load the garden credentials into the environment
set -a; . .do-it/bench-env/alaya-api.env; set +a

# foreground (both ship-gate benches, sequential single-process):
node scripts/run-full-bench-v0311.mjs

# OR detached (survives the shell), recommended for the long 500q run:
setsid node scripts/run-full-bench-v0311.mjs > /tmp/v0311-full.log 2>&1 < /dev/null &
disown
```
Useful flags (see the script header for the full list): `--bench lme-s` (single
bench), `--limit 50` (smoke, NOT release-grade), `--resume` (skip benches whose
`<log-root>/<id>.done` sentinel exists — note: no per-question intra-step resume,
so a mid-run crash re-runs the whole 500q step), `--dry-run`.

The script runs the two benches strictly sequentially so two daemons never
coexist; it was tuned for the 7.6 GB box, so a larger host only runs faster.
Per-step logs land under `var/bench-logs/v0.3.11/`.

## Ship-gate matrix (Tier-1, from `kpi-targets.md`)
| KPI | bench | size | embedding | target |
|-----|-------|------|-----------|--------|
| **K1.1** | LongMemEval-S | 500 | off | **R@5 ≥ 90%** |
| **K1.4** | LoCoMo | 1982 | off | **R@5 ≥ 55%** |

(K1.2 lme-multiturn / K1.3 lme-crossquestion ≥ 90% and K1.5 LoCoMo embedding-ON
≥ 90% are a later v0.3.11 wave; this script orchestrates K1.1 + K1.4.)

## Read the result
Each bench writes a fresh archive under `docs/bench-history/<root>/<slug>/` with
a `kpi.json`. Confirm `recall.r_at_5` (or the equivalent KPI field) against the
targets above, and that the run is clean (`llm_calls`/`offline_fallbacks` as
expected, `seed_extraction_path = official_api_compile`). Exit code 0 means
every requested bench produced a fresh archive with `kpi.json`.

## Cost (operator-gated)
With the garden key set, any extraction cache MISS is filled by a **paid**
garden LLM call (`gpt-5.4-mini`). The tracked cache covers ~100q; the full
500q LongMemEval-S run pays to fill the rest (the historical full fill was
~72,277 turns). This spend is YOUR decision — there is no autonomous billing.
If you only want a free sanity pass, run `--limit 100` against the cached
window (not release-grade).

## After the gate
- **K1.1 ≥ 90% and K1.4 ≥ 55%** → the gate passes; you may tag `v0.3.11`.
- **A KPI misses** → do NOT tune a constant to bounce the number; pull the
  per-plane / per-hop / miss-bucket diagnostics from the archive and find the
  holistic root cause first (this is a hard project rule). The recall fan-in,
  forgetting lifecycle, and reconciliation are all live and reviewed, so a miss
  is a recall-quality signal, not a wiring gap.

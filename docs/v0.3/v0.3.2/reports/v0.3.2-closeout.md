# v0.3.2 Closeout Report

## Status

CLEARED. v0.3.2 closed as a patch-internal memory-quality release on
2026-05-13. The release adds fixture-level recall evidence packs,
schema-aware candidate-signal validation inside `raw_payload`, and
write-path hardening that prevents invalid schema-grounded signals from
silently creating durable memory.

## Scope Compliance

- Public MCP tool names, descriptions, request schemas, and response
  schemas are unchanged.
- Protocol zod schemas, EventLog payload schemas, runtime config schemas,
  and storage migrations are unchanged.
- Schema-aware extraction metadata remains an internal raw-payload
  convention on `CandidateMemorySignal.raw_payload`.
- Garden/provider/host-worker output remains candidate signal input;
  Alaya still decides through deterministic triage, materialization
  routing, governance, and EventLog audit.

## Completed

- Added `packages/core/src/recall-evidence-pack.ts` for internal recall
  evidence packs and fixture-level metrics.
- Added `packages/soul/src/garden/schema-grounding.ts` for object
  detection, field candidates, field-value extraction, and validation
  metadata inside existing signal `raw_payload`.
- Routed `OfficialApiGardenProvider`, `LocalHeuristics`, daemon
  `POST_TURN_EXTRACT`, and host-worker `garden.complete_task` candidate

- Hardened `SignalService` and `MaterializationRouter` so invalid
  schema-grounded field/value payloads are deferred before memory / claim

- Added read/write integration fixtures in the daemon test suite.
- Bumped workspace packages `0.3.1` -> `0.3.2`.
